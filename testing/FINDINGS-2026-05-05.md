## 3. Hardening items (non-blocking)

### 3.1 No security headers on the frontend — *Medium*

**Status:** ❌ FAIL on TC-SEC-021
**Evidence:** `Invoke-WebRequest GET /login` returned no `Strict-Transport-Security`, `X-Frame-Options`, `X-Content-Type-Options`, `Referrer-Policy`, `Permissions-Policy`, or CSP headers.
**Risk:** Vulnerable to first-visit downgrade, clickjacking, MIME sniffing, referrer leakage.
**Fix (5 min):** Add to `apps/web/next.config.mjs`:

```js
async headers() {
  return [{
    source: '/:path*',
    headers: [
      { key: 'Strict-Transport-Security', value: 'max-age=63072000; includeSubDomains; preload' },
      { key: 'X-Frame-Options', value: 'DENY' },
      { key: 'X-Content-Type-Options', value: 'nosniff' },
      { key: 'Referrer-Policy', value: 'strict-origin-when-cross-origin' },
      { key: 'Permissions-Policy', value: 'camera=(), microphone=(), geolocation=()' },
    ]
  }]
}
```

Alternative: configure in Cloudflare Dashboard → Rules → Transform Rules → Modify Response Header.

### 3.2 `generate-captions` parses body before checking JWT — *Low*

**Status:** ⚠️ Behavioural quirk (not a security failure)
**Evidence:**
- Empty body, no auth → `400 {"error":"job_id_required"}` (reveals field name)
- Valid body, no auth → `401 {"error":"unauthorized"}` ✅

**Risk:** Lets an unauth caller probe the field schema and forces server-side JSON parsing before rejecting. JWT enforcement itself works.
**Fix (2 min):** In [supabase/functions/generate-captions/index.ts](supabase/functions/generate-captions/index.ts), move the `if (!isCron) { validateJWT... }` block above the `req.json()` parse + `job_id_required` check.

### 3.3 `llm_usage_events` only has a SELECT policy — *Info, by design*

**Status:** ⚠️ Anomaly compared to other org-scoped tables, but **intentional**
**Evidence:** Has `org_read_own (SELECT, qual: org_id = current_org_id())`. No INSERT/UPDATE/DELETE policy.
**Why it's OK:** Only Edge Functions write telemetry to it via the service-role client, which bypasses RLS. No user-facing mutation path exists.
**Action:** None needed. If you ever expose user writes, add `org_isolation_insert` / `_update` / `_delete` policies to match the pattern used on the other 16 tables.

### 3.4 Missing indexes on hot columns — *Performance, Low*

**Status:** ⚠️ Currently fast (153 signals, 5 jobs, 11 prospects) — will degrade with scale
**Missing:**

| `generation_jobs` | `status` | `poll-generation-jobs` cron filters `WHERE status='processing'` |
| `llm_usage_events` | `created_at` | `/settings/usage` charts |
| `icp_enrichment_runs` | `user_id` (FK) | Per-Supabase-advisor |

| `prospects` | `last_campaign_id` (FK) | Per-Supabase-advisor |
| `signals` | `feed_config_id` (FK) | Per-Supabase-advisor |

**Fix:** One migration with 7 `CREATE INDEX CONCURRENTLY` statements.

## 4. Informational (nice-to-have)

| Item | Detail |
|---|---|
| `current_org_id()` mutable `search_path` | Not `SECURITY DEFINER`, so risk is very low. Tighten in next migration: `ALTER FUNCTION public.current_org_id() SET search_path = public, pg_temp;` |
| `pg_net` extension installed in `public` schema | Supabase advisor recommends `extensions` schema. Move requires updating any code referencing `pg_net.*` first. |
| Several RLS policies call `auth.*()` per-row instead of `(select auth.*())` | Performance only, not security. Relevant at 10k+ rows per table. |
| Local Windows build error (paging file) | Cosmetic — affects local `next build` only. Cloudflare Pages builds succeed. |
| `typescript.ignoreBuildErrors=true` in next config | Worth turning off after the demo and fixing remaining type errors. |
